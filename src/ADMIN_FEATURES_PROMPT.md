## FEATURE 1 — Staff Management

### 1.1 Staff List View
- Show all staff in a clean table: Name, Phone, Role, Status (Active/Inactive), Created date, Last login
- Filter by role (All / Admin / Waiter / Kitchen / Cashier / Custom roles)
- Search by name or phone
- Sort by name, role, or created date

### 1.2 Add Staff
- Modal form: Name (required), Phone (required), Email (required for login), Password (auto-generated or manual), Role (dropdown of all roles from `roles` table)
- On submit: create Supabase Auth user via `supabaseAdmin.auth.admin.createUser()`, then insert into `staff_users` table linking `auth_id`
- Show the generated password once on screen with a "Copy" button (not stored anywhere after that)

### 1.3 Edit Staff
- Edit name, phone, role assignment
- Toggle active/inactive (`is_active` field) — inactive staff cannot log in (middleware already checks this)
- Cannot delete admin's own account

### 1.4 Reset Password
- "Reset Password" button per staff row — generates a new password and shows it once, or sends a reset email via Supabase Auth `resetPasswordForEmail()`

### 1.5 Delete Staff
- Soft delete (set `is_active = false`) not hard delete — preserve order history
- Confirmation dialog required

## FEATURE 2 — Role & Privilege Management

### 2.1 Roles List
- Show all roles (seeded base roles: admin, waiter, kitchen, cashier + any custom roles admin created)
- Each role card shows: role name, number of staff assigned, list of permissions granted, edit/delete buttons
- Base roles (is_custom = false) cannot be deleted, only their permissions can be adjusted

### 2.2 Create Custom Role
- Form: Role name (text input)
- Permission selector: a checklist of all 13 permissions from the `permissions` table displayed with human-readable labels:
  - `view_orders` → "View Orders"
  - `edit_orders` → "Edit Orders"
  - `confirm_orders` → "Confirm Orders"
  - `view_kitchen_queue` → "View Kitchen Queue"
  - `update_prep_status` → "Update Prep Status"
  - `view_billing` → "View Billing"
  - `generate_bills` → "Generate Bills"
  - `edit_menu` → "Edit Menu"
  - `manage_staff` → "Manage Staff"
  - `manage_roles` → "Manage Roles"
  - `view_revenue` → "View Revenue & Analytics"
  - `export_data` → "Export Data (Excel)"
  - `manage_tables` → "Manage Tables"
- On save: insert into `roles` (is_custom = true), then insert into `role_permissions` for each checked permission
- Validation: role must have at least one permission, name must be unique

### 2.3 Edit Role Permissions
- Same checklist UI, pre-filled with current permissions
- Save updates `role_permissions` (delete existing rows for this role, re-insert checked ones)
- Show a warning if removing a permission that some staff currently rely on

### 2.4 Delete Custom Role
- Only allowed if no active staff are assigned to this role
- If staff are assigned, show error: "X staff members are assigned to this role. Reassign them before deleting."

## FEATURE 3 — Real-Time Table Management

### 3.1 Table Grid View (main view)
- Visual grid of all tables as cards (like a floor plan), each showing:
  - Table number (large, centered)
  - Status badge: Empty (green) / Occupied (gold) / Needs Bill (red) / Reserved (blue)
  - Assigned waiter name
  - If occupied: customer name + order time elapsed (e.g. "45 min")
  - If Needs Bill: total bill amount
- Real-time updates via Supabase Realtime on `orders` table — status changes reflect instantly, no manual refresh

### 3.2 Table Detail Drawer
- Clicking any occupied table opens a side drawer showing:
  - Customer name + phone number
  - Full order details: each item, qty, price, notes
  - Order status timeline (Placed → Confirmed → Preparing → Ready → Served → Billed) with timestamps
  - Assigned waiter name
  - Total amount
  - Action buttons: "Mark Served", "Generate Bill", "Reassign Waiter"

### 3.3 Create / Edit Tables
- Admin can add a new table (auto-assigns next table number, or manual input)
- Edit: change table number or assigned waiter
- Delete: only if table is currently Empty

### 3.4 QR Code Generator per Table
- Each table card has a "QR Code" button
- Opens a modal showing the QR code (generated by `qrcode.react`) encoding the exact customer ordering URL: `{NEXT_PUBLIC_SITE_URL}/MenuCard?table={restaurant_tables.id}`
- "Download QR (PNG)" button exports the QR as a PNG file named `table-{table_no}-qr.png`
- "Print QR" button opens a print-optimized view of just the QR code + "Table {N}" label below it

### 3.5 Bulk QR Export
- "Download All QRs" button in the table management header generates and zips all table QR codes into a single downloadable ZIP file (use `jszip` library)

## IMPLEMENTATION NOTES

- All data mutations must go through Server Actions (never direct Supabase client mutations from the browser for write operations)
- All reads that need realtime updates use the `supabase` anon client with Supabase Realtime subscriptions on the client side
- All admin-level reads that bypass RLS (analytics aggregation, staff management, full order history) use `supabaseAdmin` (service_role) in Server Actions only
- Add the new `staff_activity_log` and `restaurant_settings` tables to Supabase by running the migration SQL in the SQL Editor:

```sql
create table staff_activity_log (
  id uuid primary key default gen_random_uuid(),
  staff_id uuid references staff_users(id) on delete set null,
  action text not null,
  details jsonb,
  created_at timestamptz not null default now()
);
alter table staff_activity_log enable row level security;
create policy "admin can view activity log" on staff_activity_log
  for select using (has_permission('manage_staff'));
create policy "server can insert activity log" on staff_activity_log
  for insert with check (true);

create table restaurant_settings (
  id uuid primary key default gen_random_uuid(),
  restaurant_name text not null default 'Hotel Taj Ooty',
  gst_number text,
  fssai_number text,
  address text,
  phone text,
  email text,
  website text,
  updated_at timestamptz not null default now()
);
alter table restaurant_settings enable row level security;
create policy "admin can manage settings" on restaurant_settings
  for all using (has_permission('manage_staff'));
insert into restaurant_settings (restaurant_name) values ('Hotel Taj Ooty');
```

- Install `jszip` for bulk QR ZIP export: `npm install jszip`
